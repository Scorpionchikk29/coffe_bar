## ☕ Автоматизированная система онлайн-заказа кофейни (PERN Stack)

## Промышленное full-stack веб-приложение для автоматизации обработки заказов, кастомизации напитков, операционного контроля очереди (панель бариста) и финансовой аналитики. Проект спроектирован с распределением вычислительной нагрузки между сервером приложений и реляционной СУБД, строгим разграничением прав доступа и полной валидацией бизнес-правил на всех уровнях.

## 🛠️ Архитектурный стек и технологии

- Frontend: React.js (v19), TypeScript, Tailwind CSS v4 (Mobile-First UI), React Router (v7), Recharts (модуль аналитики).
- Backend: Node.js, Express.js, TypeScript, REST API.
- СУБД и ORM: PostgreSQL, Prisma ORM, Хранимые процедуры и нативные триггеры PL/pgSQL.
- Качество кода: Prettier, ESLint.

## 💎 Ключевые технические решения и оптимизации## 1. Децентрализация вычислений (PL/pgSQL Триггеры)

Математический расчет стоимости позиций чека полностью делегирован на уровень СУБД, что исключает рассогласование данных и разгружает сервер приложений:

- Триггер trg_calculate_item_price (BEFORE INSERT) автоматически парсит JSONB-объект настроек напитка (объем, сиропы, тип молока), вычисляет сумму модификаторов по прайс-листу и фиксирует цену за единицу.
- Триггер trg_update_order_total (AFTER INSERT/UPDATE/DELETE) пересчитывает сумму всех связанных позиций и атомарно обновляет итоговую стоимость в родительском документе.

## 2. Динамическое управление рабочим графиком

Внедрена сквозная защита от заказов во внерабочее время. Администратор через интерфейс изменяет часы работы кофейни. Сервер сохраняет конфигурацию, а клиентская часть динамически накладывает ограничения min и max на поле выбора времени. Дополнительно реализован алгоритм автоматического переноса суток (если выбранный час самовывоза на сегодня уже прошел, система регистрирует заказ на завтра).

## 3. Индексное покрытие и оптимизация SQL-запросов

Для защиты системы от деградации производительности при росте объема транзакций:

- Эндпоинт истории заказов переведен на постраничную пагинацию (take / skip).
- В СУБД развернут составной индекс idx_orders_user_created (user_id, created_at DESC), снижающий сложность выборки и сортировки в личном кабинете до константной величины O(1).

## 4. Комплексная безопасность и 152-ФЗ

- Разделение сущностей: Конфиденциальные данные авторизации (users) и персональные данные (user_profiles) изолированы друг от друга связью 1:1.
- Криптография: Пароли хэшируются с использованием соли алгоритмом bcrypt.
- Каскадное удаление: Реализовано строгое соответствие регламентам 152-ФЗ. При удалении аккаунта срабатывает нативный ON DELETE CASCADE, бесследно стирая все персональные сведения клиента.

## 🚀 Инструкция по локальному развертыванию## Требования

- Среда выполнения Node.js (v20 или выше)
- Активный модуль PostgreSQL в Open Server Panel

## Порядок запуска

1.  Настройка окружения сервера:
    В директории server/ создайте файл .env со следующим содержимым:

DATABASE_URL="postgresql://postgres:root@localhost:5432/coffe_bar_db?schema=public"
JWT_SECRET="СлучайнаяСтрокаДляПодписиТокенов"

2.  Инициализация базы данных:
    Импортируйте структуру и справочники из файла database/coffe_bar_db.sql в вашу локальную СУБД PostgreSQL. Внутри папки server/ выполните генерацию типов ORM:

npx prisma generate

3.  Параллельный запуск приложений:
    Запустите два независимых окна терминала в вашей IDE.

- Терминал 1 (Сервер):
  cd server
  npm run dev
- Терминал 2 (Клиент):
  cd client
  npm run dev

## Фронтенд кофейни развернется по адресу http://localhost:5173, бэкенд будет принимать API-запросы на http://localhost:5000.

## 🧪 Интеграционное тестирование

Для проверки атомарности транзакций и корректности работы триггеров базы данных разработан автоматический тест. Скрипт программно создает изолированные тестовые записи, отправляет сложный заказ с кастомизацией и нулевой стоимостью, сверяет результаты расчетов СУБД с ожидаемыми бизнес-лимитами и производит каскадную очистку таблиц.
Выполнение теста (из директории server/):

npx tsx src/test-integration.ts
